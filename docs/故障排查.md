```bash
ls -ld /root/B1KSceneSmithShadow/b1k_drake_assets
ls -ld /root/B1KSceneSmithShadow/omnigibson_dataset
ls -ld /root/B1KSceneSmithShadow/patch-of-behavior-1k
ls -lh /root/B1KSceneSmithShadow/data/b1k_text_clip_index_full8662.npz
```

若任一软链接失效，先修复链接目标再运行。

## 2. 标准启动命令
```bash
bash /root/B1KSceneSmithShadow/scripts/run_shadow_main.sh \
  "A compact bedroom with one bed and one nightstand." \
  /root/B1KSceneSmithShadow/runs/bedroom_shadow_main_005
```

## 3. 常见问题与处理

### 3.1 端口冲突（7006/7007/7008）
现象：服务启动失败，日志里出现端口被占用。

处理：
```bash
for p in 7006 7007 7008; do fuser -k ${p}/tcp 2>/dev/null || true; done
pkill -f '/root/SmithPlusOmnigibson/src/optimized_server.py' || true
pkill -f 'articulated_retrieval_server|materials_retrieval_server|standalone_server.py' || true
```

### 3.2 embeddings 或 descriptions 未生效
现象：检索仍走默认配置，或找不到索引文件。

处理：
```bash
echo "$B1K_EMBEDDINGS"
echo "$B1K_DESCRIPTIONS"
```
确认 `run_shadow_main.sh` 导出的值正确，且文件存在。

### 3.3 full8662 索引读取失败
现象：`np.load(...)` 相关异常。

处理：确认主项目以下改动仍在：
- `/root/SmithPlusOmnigibson/src/custom_retrieval.py` 使用 `np.load(..., allow_pickle=True)`

### 3.4 流程中断但前半段有产物
现象：有 `house_layout.json`，但没有最终 `omnigibson_scene(_preserve).json`。

处理顺序：
1. 看 `scene.log` 最后 100 行。
2. 检查端口冲突。
3. 检查软链接目标是否存在。
4. 检查 `B1K_EMBEDDINGS` 对应 npz 是否可读。

## 4. 产物核对命令
```bash
find /root/B1KSceneSmithShadow/runs/bedroom_shadow_main_005 -maxdepth 3 -type f | \
  rg 'omnigibson_scene|shadow_manifest|scene.log|house_layout.json|resolved_config.yaml'
```

## 5. 最小回归标准
满足以下 3 条即可认为一次 run 基本可用：
1. `resolved_config.yaml` 存在。
2. `scene_000/scene.log` 存在且无持续报错。
3. 至少生成 `house_layout.json`，最好同时生成 `omnigibson_scene(_preserve).json`。
