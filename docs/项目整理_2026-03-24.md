# B1KSceneSmithShadow 项目整理（2026-03-24）

## 1. 项目定位
`/root/B1KSceneSmithShadow` 是一个外挂工作区（非 git 仓库），目标是：
- 不改 `/data/scenesmith` 核心源码，尽量通过脚本 + 环境变量把 B1K 检索接入 SceneSmith 流程。
- SceneSmith 侧使用 B1K 代理资产完成布局/物理。
- 后续可在 OmniGibson 侧按 manifest 还原。

## 4. 目录职责
- `configs/shadow_proxy.env.example`：Shadow 环境变量模板
- `shadow_proxy/asset_proxy.py`：Shadow 检索代理核心
- `scripts/`：构建索引、运行主流程、导出/重建 manifest
- `data/`：embedding 与示例 manifest
- `runs/`：每次执行输出
- `bin/bwrap`：本地 shim（用于规避特定权限报错）

## 5. 软链接依赖（关键）
当前这三个是软链接，不在本目录内：
- `b1k_drake_assets -> /root/SmithPlusOmnigibson/b1k_drake_assets`
- `omnigibson_dataset -> /root/.omnigibson`
- `patch-of-behavior-1k -> /root/sgpromax/patch-of-behavior-1k`

若这些目标路径变化，Shadow 会直接失效。

## 6. 与主项目的耦合点（已改动）
虽然 Shadow 目标是不改主仓，但当前实际上改到了：
- `/root/SmithPlusOmnigibson/src/server_launcher.py`
- `/root/SmithPlusOmnigibson/src/optimized_server.py`
- `/root/SmithPlusOmnigibson/src/custom_retrieval.py`

用途：让 `B1K_EMBEDDINGS/B1K_DESCRIPTIONS` 真正传递并加载（含 `allow_pickle=True`）。

## 7. 建议开发基线（你下次接着做）
1. 先跑一次固定入口（避免参数漂移）：
```bash
bash /root/B1KSceneSmithShadow/scripts/run_shadow_main.sh \
  "A compact bedroom with one bed and one nightstand." \
  /root/B1KSceneSmithShadow/runs/bedroom_shadow_main_005
```
2. 立即核对关键产物：
```bash
find /root/B1KSceneSmithShadow/runs/bedroom_shadow_main_005 -maxdepth 3 -type f | \
  rg 'omnigibson_scene|shadow_manifest|scene.log|house_layout.json|resolved_config.yaml'
```
3. 若仍中断，优先查：
- `scene.log`
- 7006/7007/7008 端口占用
- `optimized_server.py` 是否成功加载 full8662 embedding
